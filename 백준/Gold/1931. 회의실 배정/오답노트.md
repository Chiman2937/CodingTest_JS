# 회의실 배정 (BOJ 1931) 오답노트

## 문제 요약
N개의 회의 각각에 시작·종료 시간이 주어질 때, 한 회의실에서 **겹치지 않게 배정할 수 있는 회의의 최대 개수**를 구한다.
한 회의가 끝나는 시각과 다음 회의가 시작하는 시각이 같으면 연속 배정이 가능하다.

---

## 1차 시도 — DFS 완전 탐색 (시간 초과)

그리디로 해결이 불가능하다고 판단했다.
"지금 선택한 회의가 최선인지 미래를 보기 전에 알 수 없다"고 생각했기 때문이다.
그래서 DFS로 모든 경우를 탐색했다.

```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
let input = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(/\r?\n/);

let [n, ...meetings] = input;
meetings = meetings.map((v) => v.split(' ').map(Number));

function solution(n, meetings) {
  function dfs(index, lastTime, count) {
    if (index === n) return count;
    let result = count;
    for (let i = index + 1; i < n; i++) {
      if (lastTime < meetings[i][0]) {
        result = Math.max(result, dfs(i, meetings[i][1], count + 1));
      }
    }
    return result;
  }
  return dfs(-1, 0, 0);
}

console.log(solution(+n, meetings));
```

### 한계 / 분석
각 회의마다 "선택 / 건너뜀" 두 가지 분기가 존재하므로 시간복잡도는 **O(2^N)**.
N ≤ 100,000이면 2^100000은 실질적으로 실행 불가능하다.

---

## 2차 시도 — 그리디 + 종료 시간 기준 정렬 (오답)

문제 분류를 확인 후 그리디 문제임을 파악했다.
**종료 시간이 빠른 회의를 먼저 배정**하면, 이후 남은 시간이 가장 많아진다는 그리디 근거를 세웠다.

```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
let input = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(/\r?\n/);

let [n, ...meetings] = input;
meetings = meetings.map((v) => v.split(' ').map(Number));

function solution(n, meetings) {
  meetings.sort((a, b) => a[1] - b[1]);
  let result = 0;
  let currentTime = 0;
  for (const [s, f] of meetings) {
    if (currentTime <= s) {
      currentTime = f;
      result++;
    }
  }
  return result;
}

console.log(solution(+n, meetings));
```

### 한계 / 분석
종료 시간이 같은 회의가 여러 개 있을 때 **시작 시간 순서가 보장되지 않는다**.

예를 들어 `[0, 2]`, `[2, 2]` 두 회의가 있을 때, `[2, 2]`가 먼저 오면 `currentTime=2`가 되어
이후 `[0, 2]`는 `s=0 < currentTime=2`이므로 배정이 불가능하다.

```
[2, 2] 먼저:  currentTime=0 → [2,2] 배정(0<=2), currentTime=2 → [0,2] 배정 불가(2>0) → count=1
[0, 2] 먼저:  currentTime=0 → [0,2] 배정(0<=0), currentTime=2 → [2,2] 배정(2<=2) → count=2
```

올바른 결과는 2이므로, 종료 시간이 같을 때 **시작 시간이 작은 회의를 먼저** 배정해야 한다.

## 3차 시도 — 종료 시간 → 시작 시간 이중 정렬 (정답)

### 핵심 아이디어
정렬 기준을 **종료 시간 오름차순, 동률이면 시작 시간 오름차순**으로 변경한다.
이렇게 하면 시작·종료 시간이 같은 회의(길이 0인 회의)도 올바른 순서에 놓인다.

```
정렬 예시 (종료↑, 시작↑):
[0, 0] → [0, 1] → [1, 2] → ...

currentTime=0
  [0,0]: 0<=0 → 배정, currentTime=0, count=1
  [0,1]: 0<=0 → 배정, currentTime=1, count=2
  [1,2]: 1<=1 → 배정, currentTime=2, count=3
  ...
```

### 변수 설계
| 변수 | 역할 |
|---|---|
| `currentTime` | 마지막으로 배정된 회의의 종료 시각 |
| `result` | 배정된 회의 수 |

### 그리디 근거
종료 시간이 가장 빠른 회의를 먼저 선택하면 **남은 시간이 최대**가 된다.
이는 "활동 선택 문제(Activity Selection Problem)"의 고전적 그리디 증명과 동일하다.

### 최종 코드

```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
let input = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(/\r?\n/);

let [n, ...meetings] = input;
meetings = meetings.map((v) => v.split(' ').map(Number));

function solution(n, meetings) {
  meetings.sort((a, b) => a[1] - b[1] || a[0] - b[0]);
  let result = 0;
  let currentTime = 0;
  for (const [s, f] of meetings) {
    if (currentTime <= s) {
      currentTime = f;
      result++;
    }
  }
  return result;
}

console.log(solution(+n, meetings));
```

---

## 복잡도 비교

| 방식 | 시간복잡도 | 비고 |
|---|---|---|
| 1차 (DFS 완전 탐색) | O(2^N) | N=100,000에서 실행 불가 |
| 2차 (그리디, 단일 정렬) | O(N log N) | 정렬 후 선형 순회, 동률 케이스 오답 |
| 3차 (그리디, 이중 정렬) | O(N log N) | 동일한 복잡도, 코너케이스 통과 |

N ≤ 100,000이므로 O(N log N) 풀이가 필요하다.

---

## 핵심 교훈

1. **"그리디가 불가능하다"고 단정하기 전에 최적 부분 구조를 확인하라.**
   "종료가 빠른 것을 먼저 선택하면 남은 시간이 최대가 된다"는 근거가 성립하므로 그리디가 적용 가능했다.

2. **이중 정렬 키는 코너케이스를 잡는 안전장치다.**
   1차 키가 같을 때 2차 키를 어떻게 설정하느냐가 경계 조건을 결정한다.
   그리디 정렬 문제에서 동률 처리를 항상 명시적으로 고려할 것.

3. **`sort((a, b) => a[1] - b[1] || a[0] - b[0])` 패턴을 기억하라.**
   JS에서 `||`는 앞이 0(falsy)일 때만 뒤를 평가하므로, 다중 키 정렬에 깔끔하게 활용할 수 있다.
