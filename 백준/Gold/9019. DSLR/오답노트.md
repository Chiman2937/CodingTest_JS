# DSLR (BOJ 9019) 오답노트

## 문제 요약

4자리 숫자 A를 4자리 숫자 B로 변환하는 **최소 명령어 시퀀스**를 구하는 문제.
사용 가능한 명령어는 D, S, L, R 4가지이며, 각 명령어는 수에 특정 연산을 적용한다.
테스트 케이스 T개에 대해 각각의 최소 명령어 문자열을 출력한다.

---

## 1차 시도 — BFS + 명령어 문자열 직접 저장 (메모리 초과)

```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
let input = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(/\r?\n/);

let = [n, ...cases] = input;
cases = cases.map((v) => v.split(' ').map(Number));

function solution(cases) {
  let result = '';
  for (let [start, target] of cases) {
    let queue = [[start, '']];
    let visitors = new Set();
    let dirs = ['D', 'S', 'L', 'R'];
    while (queue.length) {
      let [n, command] = queue.shift();
      if (n === target) {
        result += command + '\n';
      }
      for (let c of dirs) {
        let v = calc(c, n);
        if (!visitors.has(v)) {
          visitors.add(v);
          queue.push([v, command + c]);
        }
      }
    }
  }
  return result;
}

function calc(command, n) {
  switch (command) {
    case 'D':
      return (n * 2) % 10000;
    case 'S':
      return n === 0 ? 9999 : n - 1;
    case 'L':
      let arr1 = String(n).padStart(4, 0).split('');
      arr1.push(arr1.shift());
      return +arr1.join('');
    case 'R':
      let arr2 = String(n).padStart(4, 0).split('');
      arr2.unshift(arr2.pop());
      return +arr2.join('');
  }
}

console.log(solution(cases));
```

### 한계 / 분석

각 큐 항목이 `[현재값, 명령어_문자열]` 쌍으로 저장되어, **명령어 문자열이 노드마다 독립 복사본**으로 누적된다.

예를 들어 최단 경로가 `LLLL...` (L이 N번)인 경우, 큐에는 아래와 같이 쌓인다:

```
[[start, ''], [v1, 'L'], [v2, 'LL'], [v3, 'LLL'], ..., [vN, 'L*N']]
```

명령어 문자열의 길이가 최대 수백 자에 달하고, 가능한 상태가 최대 10,000개이며,
테스트 케이스가 최대 10개이므로 **큐 전체의 문자열 총 길이는 O(10000 × 최대명령어길이)**에 달한다.
또한 `queue.shift()`는 O(n) 연산이라 큐 자체도 비효율적이다.

## 2차 시도 — BFS + prev 역추적 (성공)

### 핵심 아이디어

명령어 문자열을 큐에 직접 저장하는 대신, **`prev[v]` 배열에 "v에 도달하기 직전 상태와 명령어"만 저장**한다.
BFS 완료 후 목표값에서 출발 값까지 `prev`를 역추적하며 명령어를 재조립한다.

```
prev 배열 (인덱스 = 현재 값)
┌───────┬────────────────────────────┐
│ index │ prev[index]                │
├───────┼────────────────────────────┤
│ start │ -1  (시작점 표시)          │
│  v1   │ [start, 'L']              │
│  v2   │ [v1, 'L']                 │
│  v3   │ [v2, 'L']                 │
│  ...  │ ...                        │
└───────┴────────────────────────────┘

역추적: target → vN-1 → ... → v1 → start
명령어:           L         L         L
재조립(앞에 붙이기): 'L' → 'LL' → 'LLL' → ...
```

명령어 문자열을 10,000개 복사하는 대신, **각 노드당 [이전값, 명령어] 쌍 하나만** 저장하므로
메모리가 O(10000)으로 고정된다.

또한 배열 인덱스로 head 포인터를 이동하는 방식(`queue[head++]`)으로 `shift()` O(n)을 O(1)로 개선했다.

### 변수 설계

| 변수 | 역할 |
|---|---|
| `prev[v]` | v에 처음 도달했을 때의 `[이전값, 명령어]`. 미방문은 `null`, 시작점은 `-1` |
| `queue` | BFS 큐. 숫자값만 저장 |
| `head` | 큐의 현재 처리 위치 (shift 대신 포인터 이동) |

### 핵심 로직 흐름

```
1. prev[start] = -1 로 시작점 표시
2. BFS: 큐에서 n을 꺼내 4가지 명령 적용
   → prev[v] === null 이면 미방문 → prev[v] = [n, c], 큐에 추가
   → n === target 이면 break
3. 역추적: i = target
   → prev[i] === -1 이면 종료
   → [v, c] = prev[i] → command = c + command → i = v
```

### 최종 코드

```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
let input = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(/\r?\n/);

let = [t, ...cases] = input;
cases = cases.map((v) => v.split(' ').map(Number));

function solution(cases) {
  let result = '';
  for (let [start, target] of cases) {
    let prev = new Array(10000).fill(null);
    prev[start] = -1;
    let queue = [start];
    let dirs = ['D', 'S', 'L', 'R'];
    let head = 0;
    while (queue.length > head) {
      let n = queue[head++];
      if (n === target) break;
      for (let c of dirs) {
        let v = calc(c, n);
        if (prev[v] === null) {
          queue.push(v);
          prev[v] = [n, c];
        }
      }
    }
    let command = '';
    let i = target;
    while (true) {
      if (prev[i] === -1) break;
      let [v, c] = prev[i];
      command = c + command;
      i = v;
    }
    result += command + '\n';
  }
  return result;
}

function calc(command, n) {
  switch (command) {
    case 'D':
      return (n * 2) % 10000;
    case 'S':
      return n === 0 ? 9999 : n - 1;
    case 'L':
      return (n % 1000) * 10 + Math.floor(n / 1000);
    case 'R':
      return (n % 10) * 1000 + Math.floor(n / 10);
  }
}

console.log(solution(cases));
```

## 복잡도 비교

| 방식 | 공간복잡도 | 시간복잡도 | 비고 |
|---|---|---|---|
| 1차 — 문자열 직접 저장 | O(10000 × L) | O(10000 × L) | L = 최대 명령어 길이, shift()가 O(n) |
| 2차 — prev 역추적 | O(10000) | O(10000) | 상태 공간이 0~9999로 고정 |

상태 공간이 최대 10,000개로 제한되어 있으므로, BFS 자체는 O(10000)이 상한이다.
1차는 명령어 문자열을 노드마다 복사해 이 상한을 초과하지만,
2차는 prev 배열 하나로 역추적하여 메모리를 O(10000) 고정으로 유지한다.

> BFS에서 경로 자체를 저장해야 할 때는 큐에 전체 경로를 넣지 말고, **이전 노드 포인터(prev)를 저장한 뒤 역추적**하는 패턴을 사용한다.
