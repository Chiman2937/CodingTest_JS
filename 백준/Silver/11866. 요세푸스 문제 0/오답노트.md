# 요세푸스 문제 0 (BOJ 11866) 오답노트

## 문제 요약
N명이 원을 이루고, K번째 사람을 순서대로 제거한다.
제거된 순서를 요세푸스 순열로 출력한다.

---

## 1차 시도 — visited 플래그 (통과, 비효율)
```js
function solution(n, k) {
  let array = new Array(n)
    .fill(0)
    .map((v, i) => ({ value: i + 1, visited: false }));
  let result = [];
  let remain = n;
  let count = 1;
  let i = 0;

  while (remain > 0) {
    if (!array[i].visited && count !== k) {
      count++;
    } else if (!array[i].visited && count === k) {
      array[i].visited = true;
      count = 1;
      result.push(array[i].value);
      remain--;
    }
    i = (i + 1) % n;
  }

  return '<' + result.join(', ') + '>';
}
```

### 한계
항상 index +1씩 순회하기 때문에 제거된 사람도 계속 지나쳐야 한다.
k=n일 때 최악의 경우:
```
n번째 찾기:   n번 순회
n-1번째 찾기: n-1번 순회
...
```

→ n + (n-1) + ... + 1 = **O(n²)**

---

## 2차 시도 — 원형 연결리스트 (최적)

### 핵심 아이디어
배열 크기를 n+1로 고정하고, 인덱스로 연결 관계를 표현한다.
```
n=7일 때 초기 상태

index │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │
link  │ 0 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 1 │
```

- `array[i]` = i번 사람의 다음 사람 번호
- 원형 연결: 1→2→3→...→n→1

### 변수 설계
| 변수 | 역할 |
|---|---|
| `cur` | 제거할 사람의 직전 포인터 |
| `remain` | 남은 인원 수 (배열 크기가 안 변하므로 별도 관리) |

`cur`을 `n`으로 초기화하는 이유: 첫 번째 제거 대상은 k번째 사람이고, 순회 시작점이 1번이므로 그 직전인 n번에서 시작.

### 반복 로직
```
k-1번 이동 → cur이 제거할 사람의 직전에 위치
removed = array[cur]       // 제거할 사람
array[cur] = array[removed] // 연결 해제

before: cur → removed → next
after:  cur → next  (removed는 아무도 가리키지 않음)
```

제거 후 cur이 그대로 있으면 다음 반복에서 k-1번 이동했을 때
자동으로 다음 제거 대상의 직전으로 이동한다.

### 최종 코드
```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
const [n, k] = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(' ')
  .map(Number);

function solution(n, k) {
  const array = new Int32Array(n + 1);
  for (let i = 1; i <= n; i++) array[i] = (i % n) + 1;

  let result = [];
  let remain = n;
  let cur = n;

  while (remain > 0) {
    for (let i = 0; i < k - 1; i++) cur = array[cur]; // k번째 직전으로 이동
    const removed = array[cur];
    array[cur] = array[removed]; // 연결 해제
    result.push(removed);
    remain--;
  }

  return '<' + result.join(', ') + '>';
}

console.log(solution(n, k));
```

## 복잡도 비교

| 방식 | 시간 | 비고 |
|---|---|---|
| visited 플래그 | O(n²) | 제거된 사람도 계속 순회 |
| 원형 연결리스트 | O(n*k) | 제거된 사람은 링크에서 제외 |

N ≤ 1,000이라 두 방법 모두 통과하지만, 연결리스트가 구조적으로 더 올바른 풀이다
