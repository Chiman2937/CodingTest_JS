# 계단 오르기 (BOJ 2579) 오답노트

## 문제 요약
계단을 한 번에 1칸 또는 2칸씩 오를 수 있다.
연속된 3개의 계단을 모두 밟을 수 없고, 마지막 계단은 반드시 밟아야 한다.
각 계단에 적힌 점수의 최댓값을 구한다. (N ≤ 300, 점수 ≤ 10,000)

---

## 1차 시도 — 연속 횟수 추적 DP (오답)

```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
let input = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(/\r?\n/);

let [n, ...list] = input.map(Number);

function solution(n, list) {
  list.push(0);
  let dp1 = [[0, 0]];
  let dp2 = [[list[0], 1]];

  for (let i = 1; i < n; i++) {
    if (dp1[i - 1][1] === 0) {
      dp1.push([dp1[i - 1][0] + list[i], 1]);
    } else if (dp1[i - 1][1] === 2) {
      dp1.push([dp1[i - 1][0], 0]);
    } else {
      if (list[i + 1] > list[i] || i + 1 === n - 1) {
        dp1.push([dp1[i - 1][0], 0]);
      } else {
        dp1.push([dp1[i - 1][0] + list[i], 2]);
      }
    }
  }

  for (let i = 1; i < n; i++) {
    if (dp2[i - 1][1] === 0) {
      dp2.push([dp2[i - 1][0] + list[i], 1]);
    } else if (dp2[i - 1][1] === 2) {
      dp2.push([dp2[i - 1][0], 0]);
    } else {
      if (list[i + 1] > list[i] || i + 1 === n - 1) {
        dp2.push([dp2[i - 1][0], 0]);
      } else {
        dp2.push([dp2[i - 1][0] + list[i], 2]);
      }
    }
  }
  return Math.max(dp1[n - 1][0], dp2[n - 1][0]);
}

console.log(solution(n, list));
```

### 한계 / 분석
`[누적 점수, 연속 밟은 횟수]` 쌍을 dp에 기록하며, 다음 계단의 점수와 비교해 현재 계단을 건너뛸지 결정하는 그리디 방식이다.

**핵심 문제: 그리디로 "지금 건너뛸지"를 결정하면 최적 선택을 보장할 수 없다.**

예를 들어 계단 점수가 `[1, 100, 100, 1, 100, 100]`(n=6)일 때:
- 최적 경로: 2번, 3번, 5번, 6번 계단 밟기 → 100+100+100+100 = 400
- 이 풀이는 현재 인접한 값만 비교하므로 2번+3번을 밟는 것이 좋다는 판단을 미래 정보(5번, 6번) 없이는 내릴 수 없다.

또한 **마지막 계단을 반드시 밟아야 한다**는 조건과 그리디 판단이 충돌하는 케이스를 처리하지 못한다.

## 2차 시도 — 점화식 DP (통과)

### 핵심 아이디어
i번째 계단을 밟는 상황에서, 직전에 어떤 계단을 밟았는지를 기준으로 경우를 나눈다.

```
Case A: i-1번 계단을 건너뛰고 i번만 밟는 경우
        → i-2번까지 최댓값 + stair[i]
        → dp[i-2] + stair[i]

Case B: i-1번과 i번을 연속으로 밟는 경우 (2번 연속)
        → i-2번은 반드시 건너뛰어야 하므로 i-3번까지 최댓값 + stair[i-1] + stair[i]
        → dp[i-3] + stair[i-1] + stair[i]
```

두 경우 중 큰 값이 i번째 계단까지의 최댓값이다:

```
dp[i] = max(dp[i-2] + stair[i], dp[i-3] + stair[i-1] + stair[i])
```

`dp[i-3]`이 등장하므로 i는 3부터 시작한다. i < 3인 초기 케이스는 별도 처리한다.

**인덱스 처리: stair 앞에 0을 추가해 출발 지점을 stair[0]으로 만든다.**

```
stair: [0, s1, s2, s3, s4, ...]
         ↑ 출발(점수 없음)
index:  0   1   2   3   4
```

이렇게 하면 dp[0] = 0 (출발), dp[1] = stair[1], dp[2] = stair[1]+stair[2]가 자연스럽게 초기값이 된다.

### 변수 설계
| 변수 | 역할 |
|---|---|
| `stair[i]` | i번째 계단의 점수 (stair[0] = 0, 출발 지점) |
| `dp[i]` | i번째 계단을 밟았을 때 얻을 수 있는 최대 점수 |
| `dp[i-2] + stair[i]` | i-1번 건너뛰고 i번만 밟는 경우 |
| `dp[i-3] + stair[i-1] + stair[i]` | i-1번, i번 연속으로 밟는 경우 |

### 핵심 로직 흐름

```
stair.unshift(0)              // 출발 지점 추가
dp[0] = 0                     // 출발 (점수 없음)
dp[1] = stair[1]              // 1번 계단만
dp[2] = stair[1] + stair[2]  // 1번, 2번 연속 (n >= 2일 때)

for i = 3 to n:
  dp[i] = max(
    dp[i-2] + stair[i],                   // Case A
    dp[i-3] + stair[i-1] + stair[i]       // Case B
  )

return dp[n]
```

### 최종 코드

```js
const filePath = process.platform === 'linux' ? '/dev/stdin' : './input.txt';
let input = require('fs')
  .readFileSync(filePath)
  .toString()
  .trim()
  .split(/\r?\n/);

let [n, ...stair] = input.map(Number);

function solution(n, stair) {
  stair.unshift(0);
  let dp = new Array(n + 1).fill(0);
  dp[1] = stair[1];
  if (n >= 2) dp[2] = stair[1] + stair[2];

  for (let i = 3; i <= n; i++) {
    dp[i] = Math.max(dp[i - 2] + stair[i], dp[i - 3] + stair[i - 1] + stair[i]);
  }
  return dp[n];
}

console.log(solution(n, stair));
```

## 복잡도 비교

| 방식 | 시간복잡도 | 비고 |
|---|---|---|
| 연속 횟수 추적 그리디 DP | O(N) | 복잡도는 같지만 그리디 판단으로 오답 발생 |
| 점화식 DP | O(N) | 각 계단마다 2가지 경우만 비교하므로 O(1) 전이 |

N ≤ 300으로 제약이 작아 복잡도보다 **정확한 점화식 설계**가 관건인 문제다.
그리디로 "지금 건너뛸지"를 결정하면 안 되고, **각 위치에서 도달 가능한 경우의 수를 전부 표현한 점화식**으로 접근해야 한다.
