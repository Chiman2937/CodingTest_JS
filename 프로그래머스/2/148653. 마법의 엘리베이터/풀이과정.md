# 마법의 엘리베이터

## 문제 요약

- 버튼: ±1, ±10, ±100, ... (절댓값이 10^c 형태)
- 버튼 1번 누를 때마다 마법의 돌 1개 소모
- `storey`층에서 0층까지 가는 데 필요한 최소 마법의 돌 개수 반환
- 단, 현재 층 + 버튼값 < 0이면 이동 불가

**제한:** 1 ≤ storey ≤ 100,000,000

### 입출력 예

| storey | result |
|--------|--------|
| 16     | 6      |
| 2554   | 16     |

---

## 첫 번째 풀이

```js
function solution(storey) {
    let result = 0;
    while(storey > 0){
        const digit = String(storey).length
        const count = Math.round((storey-0.1) / 10**(digit - 1));
        result += count >= 6 ? 10 - count + 1 : count;
        storey = Math.abs(count * 10**(digit-1) - storey);
    }
    return result;
}
```

### 아이디어

가장 높은 자릿수부터 탐색.

예시: `storey = 15555`

1. `15555`를 최고 자릿수 기준으로 반올림 → `1.5555` → 반올림 → `2` → `20000`
2. `|20000 - 15555| = 4445` → 다음 storey
3. 이 과정을 storey가 0이 될 때까지 반복

**핵심 판단:** 현재 자릿수의 값이 5 이상이면 올림(+방향 버튼), 5 미만이면 내림(-방향 버튼)이 유리.

**문제점:** `Math.round`에서 정확히 1.5인 경우 2로 올림 → 그런데 `1에서 +5` (count=1, 버튼 1+5=6번)이 `2에서 -5` (count=2, 버튼 2+5=7번)보다 버튼 수가 적음. 즉 정확히 5인 경우엔 올림이 아니라 내림이 유리.

**해결:** `storey - 0.1`로 억지로 보정하여 정확히 5인 경우 내림으로 처리.
